# OpenVPN Dashboard

A Next.js + MongoDB admin panel for managing OpenVPN server users, generating `.ovpn` profiles, and monitoring bandwidth usage.

![Tech Stack](https://img.shields.io/badge/Next.js-14-black) ![MongoDB](https://img.shields.io/badge/MongoDB-Mongoose-green) ![TypeScript](https://img.shields.io/badge/TypeScript-5-blue)

---

## Features

### Admin Panel
- **User Management** — Create, disable/enable, and delete VPN users
- **OVPN Profile Generation** — Generate and download `.ovpn` client config files using EasyRSA
- **Monthly Traffic Stats** — View per-user upload/download for any month (dropdown selector)
- **Analytics Dashboard** — Bar, Area, and Line charts for yearly bandwidth trends
- **Active User Tracking** — Monthly active user counts
- **Top Users Leaderboard** — Bandwidth usage rankings
- **VPN Log Sync** — Parse OpenVPN status log and update MongoDB
- **Certificate Revocation** — Automatically revoke EasyRSA cert on user delete

### User Portal
- View own account status
- Download their `.ovpn` config file (when generated by admin)
- View own monthly bandwidth usage chart

## Setup

### 1. Clone and install

```bash
git clone <repo>
cd openvpn-dashboard
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

```env
MONGODB_URI=mongodb://localhost:27017/openvpn-dashboard
JWT_SECRET=your-very-long-random-secret-here
OVPN_OUTPUT_PATH=/root
EASY_RSA_PATH=/etc/openvpn/server/easy-rsa
OVPN_BASE_CONFIG=/etc/openvpn/server/client-common.txt
OPENVPN_STATUS_LOG=/var/log/openvpn/openvpn-status.log
ADMIN_DEFAULT_EMAIL=admin@vpn.local
ADMIN_DEFAULT_PASSWORD=Admin@1234
```

### 3. Seed admin user

```bash
npx ts-node scripts/seed.ts
```

### 4. Run development server

```bash
npm run dev
```

Open: http://localhost:3000

## OpenVPN Server Requirements

For `.ovpn` generation to work, your server needs:

- **EasyRSA** initialized at `EASY_RSA_PATH`
- **PKI** set up with a CA
- **Client config template** at `OVPN_BASE_CONFIG` (contains server IP, port, cipher, etc.)
- **Write permissions** on `OVPN_OUTPUT_PATH` for the Node.js process

### Example `client-common.txt`

```
client
dev tun
proto udp
remote YOUR_SERVER_IP 1194
resolv-retry infinite
nobind
persist-key
persist-tun
remote-cert-tls server
auth SHA256
cipher AES-256-CBC
verb 3
```

### Traffic Sync (Cron)

Add to crontab to auto-sync every 5 minutes:

```bash
*/5 * * * * curl -s -X POST -H "Cookie: token=ADMIN_JWT_TOKEN" http://localhost:3000/api/stats/sync
```

## API Routes

| Method | Route | Auth | Description |
|--------|-------|------|-------------|
| POST | /api/auth/login | Public | Login |
| POST | /api/auth/logout | Any | Logout |
| GET | /api/auth/me | Any | Current user |
| GET | /api/users | Admin | List users |
| POST | /api/users | Admin | Create user |
| DELETE | /api/users/:id | Admin | Delete user |
| PATCH | /api/users/:id | Admin | Update user |
| POST | /api/ovpn/generate/:id | Admin | Generate .ovpn |
| GET | /api/ovpn/generate/:id | Admin | Download .ovpn |
| GET | /api/stats?month&year | Admin | Monthly traffic |
| GET | /api/stats/analytics?year | Admin | Yearly analytics |
| POST | /api/stats/sync | Admin | Sync VPN logs |
| GET | /api/stats/my?year | User | Own traffic stats |
